Embedding custom claims in your token

This page demonstrates how custom claims can be used to include user defined fields in the token. To see an example, log a user in.

{{nameParts.fullname}} {{ nameParts.nickname || '(no nickname)' }}

This is the currently logged in user. The details in this card are driven by the custom claims included in the auth token. These include:

  • First Name
  • Surname
  • Nickname
  • Avatar URL
